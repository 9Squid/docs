The New Order Multileg message (`MsgType=AB`) lets you submit complex, multi-leg option strategies as a single atomic order. Supported strategies include vertical spreads, straddles, strangles, butterflies, iron condors, and covered calls. The order is composed of an order-level section (strategy parameters and net price) and a repeating leg group (one entry per leg) beginning with Tag 555 (`NoLegs`). All legs must be on the same underlying symbol.

## Order-Level Fields

These tags describe the overall strategy and apply to the order as a whole.

| Tag | Field | Required | Type | Description | Valid Values |
|---|---|---|---|---|---|
| 1 | Account | Y | String | Trading account identifier | Your assigned account number |
| 11 | ClOrdID | Y | String | Unique client-assigned order ID | Any unique string per session |
| 38 | OrderQty | Y | Qty | Number of spreads (contracts per leg) | Integer; use `0` for dollar-amount orders |
| 40 | OrdType | Y | Char | Order type | `1`=Market, `2`=Limit, `3`=Stop, `4`=Stop Limit |
| 44 | Price | C | Price | Net debit or credit for the full spread | Required when `OrdType=2` or `OrdType=4`; use negative for net credit |
| 54 | Side | Y | Char | Strategy direction | `1`=Buy, `2`=Sell |
| 55 | Symbol | Y | String | Underlying ticker symbol | Uppercase, e.g. `SPY`, `AAPL` |
| 59 | TimeInForce | Y | Char | Order duration | `0`=Day, `1`=GTC |
| 60 | TransactTime | Y | UTCTimestamp | Timestamp of order entry | `YYYYMMDD-HH:MM:SS.sss` |
| 167 | SecurityType | Y | String | Must be set to multileg | `MLEG` |

## Leg-Level Repeating Group

Each leg is defined in a repeating group. Tag 555 (`NoLegs`) specifies the total number of legs, followed by one set of leg tags per leg.

| Tag | Field | Required | Type | Description | Valid Values |
|---|---|---|---|---|---|
| 555 | NoLegs | Y | Int | Number of legs in this order | `2`–`4` |
| 600 | LegSymbol | Y | String | Option symbol for this leg | Uppercase ticker |
| 609 | LegSecurityType | Y | String | Security type for this leg | `CS`=Stock, `OPT`=Option |
| 612 | LegStrikePrice | C | Price | Strike price of this leg | Required when `LegSecurityType=OPT` |
| 613 | LegMaturityMonthYear | C | MonthYear | Expiry month/year of this leg | `YYYYMM`; required for options |
| 624 | LegSide | Y | Char | Side for this specific leg | `1`=Buy, `2`=Sell |
| 687 | LegQty | Y | Qty | Ratio quantity for this leg | Typically `1` per leg; use `2` for ratio spreads |
| 616 | LegOptionRatio | N | Float | Leg ratio for unequal-quantity spreads | Decimal |
| 623 | LegRatioQty | N | Float | Alternative leg ratio specification | Decimal |

<Note>
Set `SecurityType=MLEG` (Tag 167) at the order level for all multileg messages. Each leg uses its own `LegSecurityType` — typically `OPT` for option legs and `CS` for the stock leg in a covered call.
</Note>

## Examples

### Bull Call Spread — Buy 2 AAPL April 2026 $220/$230 Call spread at $3.50 net debit

```fix
8=FIX.4.4|35=AB|49=CLIENT1|56=BUILDMARKETS|34=10|52=20260212-14:30:00.000|
11=MLEG-20260212-001|1=ACC123456789|55=AAPL|167=MLEG|54=1|38=2|40=2|44=3.50|
59=0|60=20260212-14:30:00.000|
555=2|
600=AAPL|609=OPT|612=220.00|613=202604|624=1|687=1|
600=AAPL|609=OPT|612=230.00|613=202604|624=2|687=1|
10=089|
```

### Iron Condor — Sell 1 SPY March 2026 condor (short 490/495 call spread, short 470/465 put spread)

```fix
8=FIX.4.4|35=AB|49=CLIENT1|56=BUILDMARKETS|34=11|52=20260212-14:31:00.000|
11=MLEG-20260212-002|1=ACC123456789|55=SPY|167=MLEG|54=2|38=1|40=2|44=2.80|
59=0|60=20260212-14:31:00.000|
555=4|
600=SPY|609=OPT|612=495.00|613=202603|624=2|687=1|
600=SPY|609=OPT|612=490.00|613=202603|624=1|687=1|
600=SPY|609=OPT|612=470.00|613=202603|624=2|687=1|
600=SPY|609=OPT|612=465.00|613=202603|624=1|687=1|
10=134|
```

### Covered Call — Buy 100 shares of AAPL, sell 1 March 2026 $240 Call

```fix
8=FIX.4.4|35=AB|49=CLIENT1|56=BUILDMARKETS|34=12|52=20260212-14:32:00.000|
11=MLEG-20260212-003|1=ACC123456789|55=AAPL|167=MLEG|54=1|38=1|40=2|44=225.50|
59=0|60=20260212-14:32:00.000|
555=2|
600=AAPL|609=CS|624=1|687=100|
600=AAPL|609=OPT|612=240.00|613=202603|624=2|687=1|
10=077|
```

<Warning>
All legs in a multileg order must reference the same underlying symbol in Tag 55. Mismatched underlyings will result in a Business Message Reject (`MsgType=j`).
</Warning>

## Gateway Response

The gateway responds with an Execution Report (`MsgType=8`) acknowledging the multileg order. Fills on individual legs are reported as separate Execution Reports linked to the original `ClOrdID`. For leg-level detail on REST, use `GET /api/Blotter/OrderLegs?orderCode={code}`.
