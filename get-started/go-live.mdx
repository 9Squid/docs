Before you switch your integration to production, work through this checklist. Going live with Buildmarkets means real customer accounts, real money, and real regulatory obligations — it's worth taking the time to get each step right. This page covers everything you need to do before, during, and immediately after flipping to production.

<Warning>
  Never use sandbox API keys in production, and never point production traffic at the sandbox URL (`https://dev-tapp-api.tappengine.com`). The two environments are completely separate. Using sandbox credentials in production will result in authentication failures, and sending real customer data to sandbox is a compliance risk.
</Warning>

## Production base URL

```text
https://api.tappengine.com
```

This is the only change required in your code to switch environments — replace the sandbox base URL with this one and swap in your production credentials.

## Environment comparison

| Attribute | Sandbox | Production |
| --- | --- | --- |
| **Base URL** | `https://dev-tapp-api.tappengine.com` | `https://api.tappengine.com` |
| **KYC** | Simulated via SSN suffix | Real identity verification |
| **Orders** | Simulated fills | Live order routing |
| **ACH funding** | Instant, no real money | Real transfers, 1–3 business days |
| **API keys** | Sandbox-only credentials | Production-only credentials |
| **Customer data** | Use test data only | Real customer PII |

## Switching to production

Once you've completed the checklist, switching is a two-line change:

```bash
# Before (sandbox)
export BM_BASE="https://dev-tapp-api.tappengine.com"
export BM_KEY="your-sandbox-key"
export BM_SECRET="your-sandbox-secret"

# After (production)
export BM_BASE="https://api.tappengine.com"
export BM_KEY="your-production-key"
export BM_SECRET="your-production-secret"
```

No endpoint paths, request bodies, or response-handling code needs to change.
